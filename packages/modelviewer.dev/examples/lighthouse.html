Setting up <model-viewer> for good Lighthouse scores

The rest of the modelviewer.dev pages are optimized for code readability, but not necessarily for best page loading performance. This page, on the other hand, is designed to demonstrate best practices to optimize your Lighthouse score when using <model-viewer>.

Note the example visible on the top of the page sets the attribute reveal="interaction", which shows the poster image but does not begin to download the model until the user interacts. This keeps the large 3D model download from affecting your page load scores. This is not necessary on the second example that is "below the fold" as <model-viewer> lazy-loads by default.

You can inspect our best practices recommendations by "viewing source" on this page. Note the event listeners that cause the first interaction with the page to call dismissPoster(), which causes the 3D model to load without the user having to click on it.

This page is also designed as a way for both our users and our developers to track <model-viewer>'s progress in startup performance. From Chrome, simply open dev tools, go to the lighthouse tab and click "generate report". Currently we're seeing performance scores of roughly 97 for desktop and 92 for mobile (with some variability), but the goal is to continuously improve these.
