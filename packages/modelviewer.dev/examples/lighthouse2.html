Setting up <model-viewer> for good Lighthouse scores

Unlike the previous example, this page loads model-viewer.min.js immediately, which results in poorer lighthouse performance scores, as this bundle of JavaScript takes time to parse. We still make a strong effort to keep this bundle size down as much as possible to ensure this simple type of loading still gets decent performance.

It is still important to delay the loading of the 3D model, as including its load time in the lighthouse scores will tend to destroy them. As such the first <model-viewer> uses reveal="interaction" and calls dismissPoster() to cause loading to proceed. The second <model-viewer> does not require this as it lazy-loads by default since it is below the fold.

Note the performance is worse than before, at only 97 for desktop and 92 for mobile (shown above).
